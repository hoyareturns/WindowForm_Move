# EditProject.md

## 목적

첨부한 요구사항 문서를 기준으로 현재 프로젝트를 개선하기 위한 작업 계획을 먼저 수립한다.

별도로 첨부한 요구사항 문서가 없을 경우 기본적으로 `docs/requirements/requirements_CURRENT.docx`를 사용한다.

작업 계획은 `.codex/PLANS.md`에 작성한다.

## 실행 지시

AGENTS.md 규칙을 따르세요.

현재 코드 구조와 요구사항 문서를 분석해서 이번 작업용 `.codex/PLANS.md`를 먼저 작성해 주세요.

아직 코드는 수정하지 마세요.

PLANS.md에는 다음 내용을 포함해 주세요.

- 전체 요구사항 요약
- 단계별 작업 계획
- 각 단계의 목표
- 작업 범위
- 제외 범위
- 예상 수정 파일
- 테스트 방법
- 위험 요소
- 사용자 확인이 필요한 사항

PLANS.md 작성 후에는 내용을 보여주고 사용자 승인을 기다려 주세요.

## 주의사항

- 전체 기능을 한 번에 구현하지 마세요.
- 구현 방법은 현재 코드 구조에 맞게 Codex가 자율적으로 판단해 주세요.
- 단, 요구사항 범위를 벗어난 기능은 임의로 추가하지 마세요.
- 사용자가 승인한 단계만 구현해 주세요.
- 이미 존재하는 프로젝트 구조, 리소스 경로, 설정 방식은 우선 재사용해 주세요.
